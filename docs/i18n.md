# 🌍 Multilingual Support (i18n)

> **[🇩🇪 Deutsche Version weiter unten](#-mehrsprachigkeit-i18n--deutsche-version)**

---

## 🇬🇧 English

SpiritWorld City ships with a **lightweight, dependency-free i18n system** built around
`lib/src/core/i18n/app_strings.dart`.
No external packages are required — the system is designed to be simple and easy to extend.

### Supported Languages

| Code | Language | Status |
|------|----------|--------|
| `de` | German (Deutsch) | ✅ Default |
| `en` | English | ✅ Supported |

### How It Works

All translations live in a single `Map<String, Map<String, String>>` inside `AppStrings`:

```dart
static final Map<String, Map<String, String>> _translations = {
  'de': {
    'menu.title': 'SpiritWorld City',
    'dialog.talk': 'Sprich',
    // …
  },
  'en': {
    'menu.title': 'SpiritWorld City',
    'dialog.talk': 'Talk',
    // …
  },
};
```

The active language is stored in `AppStrings._currentLanguage` (defaults to `'de'`).

### API Reference

| Method / Property | Description |
|-------------------|-------------|
| `AppStrings.get('key')` | Returns the translated string for the current language. Falls back to the key itself if not found. |
| `AppStrings.setLanguage('en')` | Switches the active language at runtime. |
| `AppStrings.currentLanguage` | Returns the currently active language code (e.g. `'de'`). |

### Usage in Code

```dart
import 'package:spiritual_city/src/core/i18n/app_strings.dart';

// Retrieve a string
final label = AppStrings.get('dialog.talk'); // → 'Talk' or 'Sprich'

// Switch language
AppStrings.setLanguage('en');

// Use in a widget
Text(AppStrings.get('menu.newGame'))
```

> ⚠️ **Rule:** Every user-visible string **must** go through `AppStrings.get()`.
> Hard-coded strings in widgets or dialogs are not allowed.

### Key Naming Convention

Keys follow a **dot-separated namespace** pattern:

```
<screen>.<element>[.<variant>]
```

Examples:

| Key | Screen / context |
|-----|-----------------|
| `menu.newGame` | Main menu – "New Game" button |
| `difficulty.easy.desc` | Difficulty screen – easy mode description |
| `dialog.talk` | NPC dialog – Talk action chip |
| `loadGame.empty` | Load-game screen – empty state message |
| `common.language` | Shared / settings |

### Adding a New Language

1. Open `lib/src/core/i18n/app_strings.dart`.
2. Add a new entry to `_translations` with the [IETF language tag](https://en.wikipedia.org/wiki/IETF_language_tag)
   as the key (e.g. `'fr'` for French):

   ```dart
   'fr': {
     'menu.title': 'SpiritWorld City',
     'menu.newGame': 'Nouveau jeu',
     // … all keys must be present
   },
   ```

3. Make sure **every key** that exists in `'de'` and `'en'` also exists in your new locale.
   Missing keys fall back to the raw key string, which will appear in the UI.
4. Expose the new language in the Settings screen so players can select it.

### Adding a New String

1. Choose a descriptive key following the naming convention above.
2. Add the key **and** its translation to **every** locale in `_translations`.

   ```dart
   // In 'de':
   'settings.sound': 'Ton',

   // In 'en':
   'settings.sound': 'Sound',
   ```

3. Use `AppStrings.get('settings.sound')` in your widget.

## 🇩🇪 Mehrsprachigkeit (i18n) – Deutsche Version

SpiritWorld City enthält ein **schlankes, paketfreies i18n-System** auf Basis von
`lib/src/core/i18n/app_strings.dart`.
Keine externen Pakete nötig – einfach und leicht erweiterbar.

### Unterstützte Sprachen

| Code | Sprache | Status |
|------|---------|--------|
| `de` | Deutsch | ✅ Standard |
| `en` | Englisch (English) | ✅ Unterstützt |

### Funktionsweise

Alle Übersetzungen befinden sich in einer einzigen `Map<String, Map<String, String>>`
in `AppStrings`:

```dart
static final Map<String, Map<String, String>> _translations = {
  'de': {
    'menu.title': 'SpiritWorld City',
    'dialog.talk': 'Sprich',
    // …
  },
  'en': {
    'menu.title': 'SpiritWorld City',
    'dialog.talk': 'Talk',
    // …
  },
};
```

Die aktive Sprache wird in `AppStrings._currentLanguage` gespeichert (Standard: `'de'`).

### API-Referenz

| Methode / Eigenschaft | Beschreibung |
|-----------------------|--------------|
| `AppStrings.get('key')` | Gibt den übersetzten String für die aktuelle Sprache zurück. Fallback: der Schlüssel selbst. |
| `AppStrings.setLanguage('en')` | Wechselt die aktive Sprache zur Laufzeit. |
| `AppStrings.currentLanguage` | Gibt den aktuellen Sprachcode zurück (z. B. `'de'`). |

### Verwendung im Code

```dart
import 'package:spiritual_city/src/core/i18n/app_strings.dart';

// String abrufen
final label = AppStrings.get('dialog.talk'); // → 'Sprich' oder 'Talk'

// Sprache wechseln
AppStrings.setLanguage('en');

// In einem Widget verwenden
Text(AppStrings.get('menu.newGame'))
```

> ⚠️ **Regel:** Jeder sichtbare Text **muss** über `AppStrings.get()` abgerufen werden.
> Hart kodierte Strings in Widgets oder Dialogen sind nicht erlaubt.

### Schlüssel-Namenskonvention

Schlüssel folgen dem **Punkt-getrennten Namespace**-Muster:

```
<screen>.<element>[.<variante>]
```

Beispiele:

| Schlüssel | Kontext |
|-----------|---------|
| `menu.newGame` | Hauptmenü – Schaltfläche „Neues Spiel" |
| `difficulty.easy.desc` | Schwierigkeitsauswahl – Beschreibung Einfach |
| `dialog.talk` | NPC-Dialog – Aktion „Sprich" |
| `loadGame.empty` | Laden-Screen – Leerzustand |
| `common.language` | Gemeinsam / Einstellungen |

### Neue Sprache hinzufügen

1. `lib/src/core/i18n/app_strings.dart` öffnen.
2. Neuen Eintrag in `_translations` mit dem [IETF-Sprach-Tag](https://en.wikipedia.org/wiki/IETF_language_tag)
   als Schlüssel hinzufügen (z. B. `'fr'` für Französisch):

   ```dart
   'fr': {
     'menu.title': 'SpiritWorld City',
     'menu.newGame': 'Nouveau jeu',
     // … alle Schlüssel müssen vorhanden sein
   },
   ```

3. Sicherstellen, dass **jeder Schlüssel** aus `'de'` und `'en'` auch in der neuen Locale vorhanden ist.
   Fehlende Schlüssel werden als roher Schlüsselname angezeigt.
4. Die neue Sprache in den Einstellungen verfügbar machen, damit Spieler sie auswählen können.

### Neuen String hinzufügen

1. Beschreibenden Schlüssel nach der Namenskonvention wählen.
2. Schlüssel **und** Übersetzung in **jede** Locale in `_translations` eintragen:

   ```dart
   // In 'de':
   'settings.sound': 'Ton',

   // In 'en':
   'settings.sound': 'Sound',
   ```

3. `AppStrings.get('settings.sound')` im Widget verwenden.
